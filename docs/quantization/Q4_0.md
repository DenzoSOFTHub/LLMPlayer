# Q4_0 -- 4-bit Block Quant

## Overview

| Property | Value |
|----------|-------|
| Full name | 4-bit Block Quantization |
| GGML type ID | 2 |
| Bits per weight | 4.5 bpw |
| Block size | 32 elements |
| Block bytes | 18 bytes |
| Compression ratio | 7.1x vs F32 |

## Data Layout

Each 18-byte block encodes 32 weights:

```
Offset  Size   Field    Description
------  -----  -------  -------------------------------------------
0       2      d        fp16 block scale
2       16     qs       16 bytes of 4-bit nibbles (32 weights, 2 per byte)
------  -----  -------  -------------------------------------------
Total:  18 bytes
```

### Interleaved Nibble Layout

Q4_0 uses an **interleaved** nibble layout (different from Q5_0, IQ4_NL, and MXFP4 which use split layout):

```
For weight at position j:
    byte_pos = j / 2
    if j is even:  nibble = qs[byte_pos] & 0x0F       // low nibble
    if j is odd:   nibble = (qs[byte_pos] >> 4) & 0x0F // high nibble
```

This means byte `qs[i]` holds weights at positions `2*i` (low nibble) and `2*i+1` (high nibble).

## Dequantization Formula

```
value = (nibble - 8) * d
```

The zero-point is fixed at 8, giving a symmetric signed range of -8 to +7 scaled by `d`.

| Property | Value |

| Fused SIMD class | None |

| Special path | Q4_0 x Q8_0 integer dot product (avoids float dequantization) |

### Q4_0 x Q8_0 Mixed Integer Dot

When the other operand is Q8_0, a fast integer path is used:

```
for each aligned block pair:
    for i in 0..15:
        lo = (qs_q4[i] & 0x0F) - 8
        hi = ((qs_q4[i] >> 4) & 0x0F) - 8
        isum += lo * qs_q8[2*i] + hi * qs_q8[2*i + 1]
    result += d_q4 * d_q8 * isum
```

## Performance Characteristics

Q4_0 is the original 4-bit quantization format, predating the K-quant series. It uses a simple fixed zero-point (8) with a single block-level scale, without the per-sub-block scale/minimum granularity that Q4_K provides.

At the same 4.5 bpw as Q4_K, Q4_0 generally produces lower quality because:
- Only 1 scale per 32 weights (vs 1 scale + 1 min per 32 weights in Q4_K)
- Fixed zero-point of 8 (vs learned per-sub-block minimum in Q4_K)
- Smaller block size means more scale overhead but less scale sharing

The smaller 32-element blocks can be advantageous for tensors with high variance across small neighborhoods, but for most model weights, Q4_K's adaptive sub-block scales provide better reconstruction accuracy.

## Typical Usage

- Some older GGUF quantizations predating the K-quant series
- Legacy compatibility
- Q4_K_M is generally preferred over Q4_0 for new quantizations
