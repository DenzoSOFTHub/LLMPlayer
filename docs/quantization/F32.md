# F32 -- IEEE Float 32 (Single Precision)

## Overview

| Property | Value |
|----------|-------|
| Full name | IEEE 754 Single-Precision Floating Point |
| GGML type ID | 0 |
| Bits per weight | 32 bpw |
| Block size | 1 element |
| Block bytes | 4 bytes |
| Compression ratio | 1x (no compression) |

## Data Layout

Each weight is stored as a standard 32-bit IEEE 754 float:

```
Bit layout (32 bits):
[sign (1 bit)] [exponent (8 bits)] [mantissa (23 bits)]
```

No quantization or compression is applied. Direct native float access.

## Dequantization Formula

No dequantization needed:

```java
float value = data.getFloatLE(index * 4);
```

## CUDA Kernel

| Property | Value |
|----------|-------|
| Kernel file | `matmul_f32.cu` |
| Strategy | Standard float matmul |

## SIMD Optimization

| Property | Value |
|----------|-------|
| Fused SIMD class | None (direct SIMD vector operations on raw floats) |
| CPU dot path | Bulk dequantize to float array, then SIMD dot via `VectorOpsFactory.get().dot()` |

The `F32FloatTensor.dot()` implementation copies the raw float data into a temporary buffer and delegates to the SIMD dot product. Since the data is already F32, the "dequantization" is just a memory copy (needed because the data lives in memory-mapped `TensorData` rather than a plain float array).

## Performance Characteristics

F32 is the baseline format with zero quality loss. However, it is the least practical for inference:

| Model size | F32 memory | Q4_K memory | Ratio |
|------------|------------|-------------|-------|
| 1B params | ~4 GB | ~0.6 GB | 7.1x |
| 7B params | ~28 GB | ~4.0 GB | 7.1x |
| 13B params | ~52 GB | ~7.4 GB | 7.1x |
| 70B params | ~280 GB | ~40 GB | 7.1x |

The memory bandwidth requirement is also 7x higher than Q4_K, which directly translates to lower tokens/second for bandwidth-bound inference.

## Typical Usage

- Rarely used for inference (too large and too slow)
- Some GGUF files include F32 for the token embedding tensor even when other tensors are quantized
- Internal computations (activations, intermediate results) use F32 regardless of weight quantization
- Reference format for measuring quantization quality degradation
