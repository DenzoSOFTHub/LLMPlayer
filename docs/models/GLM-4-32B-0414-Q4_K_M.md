## Architecture

**Inference Engine:** `InferenceEngine` (standard transformer path)

| Property | Value |
|----------|-------|
| Normalization | Pre+Post norm (post_attn_norm + post_ffn_norm) |
| Attention | GQA with RoPE NEOX |
| FFN Layout | Packed FFN (wUp=[2*ffnDim], gate+up merged) |
| FFN Activation | SwiGLU |

**Architecture notes:**
- GLM4 uses both pre-norm and post-norm: RMSNorm before and after both the attention and FFN blocks. This is distinct from most architectures which use pre-norm only.
- FFN uses packed gate+up projection that must be split during the forward pass.

## GPU Profile

- **VRAM fit:** Minimal offload. 18.8 GB far exceeds 6 GB VRAM budget.
- **Offload mode:** First-N-layers strategy with per-tensor CUDA.
- **CudaForwardPass:** Supported (dense pre-norm with separate Q/K/V).
- **Benchmark data:** None available.

## Known Issues

- Post-norm adds extra RMSNorm operations per layer compared to standard pre-norm-only architectures, slightly increasing compute per token.
- At 18.8 GB, very few layers can be offloaded to a 6 GB GPU, making CPU-only inference the practical path.
